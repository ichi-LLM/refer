JAMAとExcelを連携させて、要件管理を効率化するツール

🐛 フィールド名が不明な場合、デバッグモードで確認できます
python main.py fetch -o debug.xlsx --debug

📥 要件の取得

JAMAプロジェクトから要件を階層構造で取得し、見やすいExcel形式で出力します。

使い慣れたExcelで要件を編集。新規追加、更新、削除が直感的に行えます。

編集した内容をJAMAに一括反映。ドライラン機能で安全に更新できます。

📥 要件の取得

目的 コマンド
プロジェクト全体 python main.py fetch -o all.xlsx

📊Excelファイルの編集方法

操作の種類

新規 JAMA_IDを空欄にして、新しい要件を追加
更新 W列（要件更新）を「する」に設定
削除 B列（メモ/コメント）に「削除」と入力

更新ロジック

A列（JAMA_ID） B列（メモ） W列（要件更新） 動作
空欄 （任意） （任意） 新規作成
あり 「削除」 （任意） 削除
あり それ以外 「する」 更新（空欄フィールドは保持）
あり それ以外 それ以外 スキップ

Description編集（5行形式）

📝 SYSPのDescriptionを詳細な5行形式のテーブルで編集できます
  1. 「Description更新」列を「する」に変更
  2. 「新Description参照」列のリンクをクリック
  3. Description_editシートで5行形式のテーブルを編集

❓よくある質問

Q: ドライランって何ですか？

A: 実際にJAMAを更新せずに、どんな変更が行われるかを確認する機能です。本番前のリハーサルのようなものです。

Q: 大量のデータを処理すると時間がかかる？

A: 10万件以上のデータでも処理できますが、Excel作成に時間がかかります。進捗表示を見ながらお待ちください。階層解析を高速化するインデックスを使用しているので、以前より大幅に高速化されています。

Q: エラーが出たらどうすれば？

A: jama_tool.logファイルに詳細なエラー情報が記録されています。それを確認してください。

🆕最新の変更点

更新ロジックの改善

  • B列を「メモ/コメント」欄に変更（自由記入）
  • W列を「要件更新」に変更（「する」で更新対象）
  • JAMA_IDとW列の組み合わせで処理を判定
  • dry-run時に更新対象を全件表示
  • 更新されるフィールドを明示

進捗表示の追加

  • Excelファイル読み込み時の進捗表示
  • 要件分類時の進捗表示
  • 更新処理時の進捗表示（50件ごと）

SYSP項目の自動処理（拡張版）

  • nameフィールドに「SYSP」を含む項目を自動検出
  • すべてのSYSP項目に対してDescriptionテンプレートを生成
  • 双方向リンク（「編集画面へ」⇔「一覧に戻る」）が正しく機能
  • 各テンプレートに要件名を表示（視認性向上）
  • SYSPアイテムの全フィールドを自動検出（Tags、Reason等）

データ保護機能

  • Excelの空欄セルは更新対象から除外
  • 既存のJAMAデータを安全に保持
  • 必要な項目のみを選択的に更新

デバッグ機能（拡張版）

  • --debugオプションでフィールド構造を確認
  • 最初のアイテムと最初のSYSPアイテムの両方を表示
  • APIレスポンスの詳細を把握可能
  • フィールド名の大文字小文字や区切り文字の違いを自動処理

⚡パフォーマンスと進捗表示

大量データ処理時の進捗表示

10万件以上のデータでも、リアルタイムで進捗を確認できます：

パフォーマンス設定

config.jsonで処理速度を調整できます：

"performance": {
    "column_width_check_rows": 100      // 列幅計算時にチェックする行数
}
