📊Excelファイルの編集方法

操作の種類

新規作成

JAMA_ID（A列）を空欄にして、必要な情報を記入

更新

要件更新（W列）を「する」に設定して、更新したい項目を編集

削除

メモ/コメント（B列）に「削除」と完全一致で入力

⚠️
削除操作の重要な注意
B列に「削除」と完全一致で入力すると、その要件は削除されます。
「削除予定」「削除したい」などは削除されません。誤入力に十分注意してください。

更新ロジック早見表

A列
(JAMA_ID)		 B列
(メモ)		 W列
(要件更新)		 動作 説明
空欄 （任意） （任意） 新規作成 新しい要件として追加
あり 削除 （任意） 削除 要件を削除（取り消し不可）
あり それ以外 する 更新 変更した項目のみ更新
あり それ以外 空欄/しない スキップ 何もしない

💻主なコマンド

📄 テンプレート作成

💡 JAMAに接続せずに、サンプルデータ入りのテンプレートを作成できます
python main.py template -o template.xlsx

📥 要件の取得

目的 コマンド
プロジェクト全体 python main.py fetch -o all.xlsx
特定の場所から（番号） python main.py fetch -o part.xlsx -s 6.1.5
特定の場所から（名前） python main.py fetch -o part.xlsx -n "公共駐車場"
階層を制限 python main.py fetch -o shallow.xlsx -d 3
件数を制限 python main.py fetch -o limited.xlsx --count 5000

📤 要件の更新

python main.py update -i requirements.xlsx --dry-run

↑ まずドライランで確認（実際の更新はしません）

python main.py update -i requirements.xlsx

↑ 問題なければ実行

📝Description編集（SYSP項目）

📋 SYSPアイテムは、専用の5行形式テーブルでDescriptionを編集できます

編集手順

  1. Requirement_of_Driverシートで、W列（要件更新）を「する」に設定
  2. X列（新Description参照）の「編集画面へ」リンクをクリック
  3. Description_editシートに移動し、5行テーブルを編集
  4. 「一覧に戻る」リンクで元のシートに戻る

5行テーブルの構造

行 内容 説明
1行目 I/O Type IN（入力）/ OUT（出力）
2行目 項目名 (a)～(d)の4項目に分類
3行目 Data Name データ名を記入
4行目 Data Label データラベルを記入
5行目 Data 実データを記入

❓よくある質問

Q: ドライランって何ですか？

A: 実際にJAMAを更新せずに、どんな変更が行われるかを事前に確認する機能です。安全に更新するための重要な機能です。

Q: 空欄のフィールドはどうなりますか？

A: 空欄のフィールドは更新されません。JAMAの既存の値がそのまま保持されます。

Q: 大量のデータを処理できますか？

A: はい、可能です。推奨上限は16,000件です。それ以上の場合は、コンポーネント単位で分割して処理することをお勧めします。

Q: エラーが出たらどうすれば？

A: jama_tool.logファイルに詳細なエラー情報が記録されています。エラーメッセージと共に確認してください。

Q: プロキシ設定は必須ですか？

A: はい、必須です。社内ネットワークからJAMAにアクセスする場合、プロキシ設定が必要です。

⚠️重要な注意事項

🔴
削除操作は取り消せません
B列に「削除」と入力した要件は、JAMAから完全に削除されます。
🔑
API Secretは一度だけ
JAMA API Secretは作成時に一度しか表示されません。必ずメモしてください。
💾
更新前のバックアップ
大量の要件を更新する前には、必ずJAMAのバックアップを取ってください。
🔄
ドライラン推奨
必ず --dry-run オプションで更新内容を確認してから実行してください。
